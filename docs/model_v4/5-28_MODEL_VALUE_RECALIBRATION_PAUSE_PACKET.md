# Model Value Recalibration Pause Packet - 2026-05-28

## Why We Are Pausing

The UI is usable enough for inspection now, but the football value layer is not trustworthy yet.
The biggest issue is not navigation; it is model behavior.

Observed examples:

- Player Board shows Keenan Allen with `private_score = 82.4` from the active data pack.
- Keenan Allen is a 34-year-old WR and should not look like a clean high-value dynasty asset.
- Darius Slayton shows `private_score = 78.88` in the active data pack despite being a fragile depth/role player.
- Trade-for candidate logic can surface players because the model score is high, even when football context says the player is not a realistic breakout or priority target.
- Rookie pick surfaces can read as if no pick is worth using and everything should be traded/deferred.

The project should remain paused until the model value layer is audited and recalibrated.

## Important Discovery

There are multiple scoring layers in the app.

### 1. Active Player Board / Active Data Pack

The Player Board currently reads from:

```text
local_exports/data_packs/lve_sleeper_20260505_pdf_ranks/model_outputs.csv
```

This file uses:

```text
model_version = veteran_lve_stats_first_v1_0_0
```

This is where the most alarming values appear:

```text
Keenan Allen private_score = 82.4
Darius Slayton private_score = 78.88
```

This layer appears too production/role-score heavy and not strict enough on:

- age cliff
- declining roster security
- real depth-chart role
- dynasty liquidation risk
- replacement-level WR availability
- market disagreement as a warning rather than an edge

### 2. Model v4 Current Value Checkpoint

The newer Model v4 current-value layer reads from:

```text
local_exports/model_v4/current_value/latest/current_player_value_review_rows.csv
```

For Keenan Allen, this layer is much more cautious:

```text
checkpoint_review_score = 41.6097
confidence_cap = 0.8
warnings = licensed route metrics unavailable, team mismatch/source review, identity cap, partial/quarantined join cap
```

So the project has a surface mismatch:

- the UI Player Board is showing old active-pack value
- newer Model v4 review layers are more conservative
- trade/pick pages may be consuming different layers depending on feature

Before tuning formulas, we need to decide which model layer is the source of truth.

## How The Project Currently Works

### Data Pack Layer

The active snapshot supplies league and player rows:

```text
local_exports/data_packs/lve_sleeper_20260505_pdf_ranks/
```

Key files:

- `model_outputs.csv`
- `fact_rosters.csv`
- `fact_official_rankings.csv`
- `fact_future_picks.csv`
- `fact_pick_values.csv`
- `dim_players.csv`

This layer powers much of the app UI.

Problem: `model_outputs.csv` is an older stats-first veteran model and may be overvaluing aging/depth veterans.

### Model v4 Evidence Layer

The Model v4 evidence spine lives under:

```text
local_exports/model_v4/evidence_matrices/latest/
```

It separates admitted factual evidence from market/projection/context fields.

Goal:

- private model value uses admitted football evidence
- market/ADP/rankings/projections stay display-only
- warnings and confidence caps explain gaps

### Current Player Value Layer

Current player values are generated in phases:

- replacement/VORP
- RB/WR value
- QB/TE value
- lifecycle/archetype
- confidence/missingness
- current-value checkpoint

Primary output:

```text
local_exports/model_v4/current_value/latest/current_player_value_review_rows.csv
```

This layer is review-only and appears more cautious than the active Player Board.

### Dynasty Asset Value Layer

This combines current players, rookies, and picks into a unified review-only value layer:

```text
local_exports/model_v4/dynasty_asset_value/latest/dynasty_asset_value_review_rows.csv
```

This is used by some newer tools:

- Startup Slot Simulator
- Trade Review
- Roster Opportunity Cost
- Pick Decision Lab

### Trade Review Layer

Trade-for candidates are generated by:

```text
src/services/model_v4_sprint14c_trade_review_service.py
```

Current logic:

```text
trade_for_pool = top 35 external current_player rows by dynasty_asset_value_review_score
```

This means "trade-for candidate" mostly means:

> high model score on another roster

It does not yet mean:

- realistic acquisition target
- breakout candidate
- role-secure target
- market-price efficient target
- actual recommendation

This label is misleading and probably should be renamed or rebuilt.

### Rookie Pick / Draft Board Layer

Rookie picks and draft board surfaces compare:

- rookie model values
- pick baselines
- startup-slot context
- defer/trade-down context

Problem:

If pick baselines are too high or rookie values too conservative, the UI can imply:

> every pick should be traded or deferred

That is not useful unless the model can prove that the rookie class is historically weak and the pick market is strong.

## Main Hypotheses To Audit

1. The app is mixing old active-pack scores with newer Model v4 scores.
2. The old active `veteran_lve_stats_first_v1_0_0` score is overvaluing past production and target volume.
3. Age/lifecycle caps are not strong enough in the active Player Board layer.
4. Depth-chart and roster-role evidence is not strict enough for WRs like Darius Slayton.
5. Trade-for candidates are mislabeled: they are high-score external assets, not actionable trade targets.
6. Market disagreement is being framed as opportunity too often instead of source/model-error warning.
7. Rookie pick baselines may be inflated relative to the rookie board, forcing every pick into trade/defer review.
8. The model lacks "replacement-level dynasty reality" for older WRs and depth WRs.

## Recommended Next Build Sprint After Limits Reset

### Phase 1 - Source-of-Truth Cleanup

Goal:
Stop the app from mixing old active data-pack value with newer Model v4 value without clear labels.

1. Add a visible model-source column to Player Board.

3. Add `legacy_formula_warning` to old `veteran_lve_stats_first_v1_0_0` rows.
4. Do not let legacy active-pack score drive trade-for candidate labels.

Goal:
Fix aging/depth veteran overvaluation.
