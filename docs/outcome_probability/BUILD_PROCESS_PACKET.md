4. v1 calibrated probability model.

## Weekly Replacement-Line Method

Implemented by `compute_weekly_replacement_lines`:

1. Select top 10 QB scores as required QB starters.
2. Select top 20 RB scores as required RB starters.
3. Select top 30 WR scores as required WR starters.
4. Select top 10 TE scores as required TE starters.
5. Remove required RB/WR/TE from the FLEX pool.
6. Select top 20 remaining RB/WR/TE as FLEX.
7. Store required lines, next-up lines, flex cut score, position final starter counts, final replacement line, and cutoff tie counts.
8. Use score thresholds for ties.

## Leakage Guardrails

Forbidden predictive/scoring/model inputs:

- ADP
- FantasyPros rankings/projections
- public rankings/projections
- public consensus
- startup rankings
- dynasty rankings
- trade calculators/trade values
- market rank
- league rank
- RotoWire projections/rankings/outlooks/values
- prior fantasy draft history
- fantasy acquisition cost
- legacy NWR `private_score`
- prior NWR model outputs
- manual hindsight notes
- end-of-season stats as features for the same season being predicted
- future team/depth-chart changes unavailable at prediction time
- post-breakout information

Sprint 1 scoring/labels may use raw stat components and the NWR scoring config. Future point-in-time features must prove:

`feature_available_timestamp <= input_snapshot_date < label_available_date`

The helper `forbidden_input_violations` detects obvious forbidden field names in proposed scoring/label inputs.

- No committed full historical raw stat component dataset is available for all player-weeks.
- No scoring calendar table exists yet beyond config scaffold.
